# 立即开始测试

## 第一步：启动服务器

**双击运行这个文件：**
```
start-local-dev.bat
```

会打开两个黑色窗口，**不要关闭它们**！

## 第二步：等待启动完成

### 窗口 1 - Local API Server
应该显示：
```
✓ 本地 API 服务器运行在 http://localhost:3001
✓ OCR 端点: http://localhost:3001/api/ocr
```

### 窗口 2 - Expo Dev Server  
应该显示：
```
› Metro waiting on exp://192.168.x.x:8081
› Scan the QR code above with Expo Go (Android) or the Camera app (iOS)

› Press w │ open web
```

**看到这些信息后，按键盘上的 `w` 键**，会自动在浏览器中打开应用！

## 第三步：测试扫描功能

1. 浏览器会自动打开应用首页
2. 点击 **"启动视觉诊断"** 按钮
3. 点击虚线框，选择一张图片（任意图片都可以）
4. 点击 **"开始扫描"** 按钮
5. 等待处理（会显示 6 个步骤）：
   - 正在处理图像... (1/6)
   - 正在识别答题卡... (2/6)
   - 正在提取答案... (3/6)
   - 正在评分... (4/6)
   - 正在分析错题... (5/6)
   - 正在生成学习路径... (6/6)
6. 自动跳转到报告页面

## 如果出现问题

### 问题：窗口显示 "Cannot find module"

**原因：** 你不在项目目录下

**解决：** 
1. 关闭所有窗口
2. 打开文件资源管理器
3. 进入项目文件夹：`D:\桌面文件\project-root\2.project-root-ai-evaluation-pro`
4. 在文件夹空白处，按住 Shift 键，右键点击
5. 选择"在此处打开 PowerShell 窗口"或"在此处打开命令提示符"
6. 双击 `start-local-dev.bat`

### 问题：浏览器没有自动打开

**解决：** 手动在浏览器中访问 `http://localhost:8081`

### 问题：显示 "Not found"

这可能是路由问题，请截图给我看：
1. 浏览器地址栏的 URL
2. 页面显示的内容
3. 浏览器控制台的错误信息（按 F12 打开）

## 查看日志

### API 服务器日志
在 "Local API Server" 窗口中可以看到：
- 收到的请求
- OCR 处理结果
- 错误信息

### Expo 开发服务器日志  
在 "Expo Dev Server" 窗口中可以看到：
- 页面加载信息
- JavaScript 错误
- 网络请求

### 浏览器控制台
按 F12 打开浏览器开发者工具，可以看到：
- 前端日志
- 网络请求
- JavaScript 错误
